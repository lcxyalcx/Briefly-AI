# Briefly AI 📚

Briefly AI 是一款面向研究生与科研工作者的本地化文献管理桌面软件。它围绕“导入 → 解析 → 理解 → 检索 → 复用”的阅读链路，帮助用户在本机完成 PDF 导入、结构化解析、RAG 问答、AI 摘要、引用追踪与阅读笔记整理。

## ✨ 核心特性

- 🖥️ 本地桌面端：基于 `Electron + React + TypeScript`
- 📥 PDF 批量导入：通过系统文件选择器导入并复制到应用私有目录
- 🧩 结构化解析：自动抽取标题、作者、摘要、章节结构、参考文献
- 🗂️ 文献管理：支持标签、阅读状态、关键词分类
- 🔎 混合检索：本地向量相似度 + 关键词覆盖
- 🧠 AI 阅读辅助：TL;DR、方法总结、创新点、实验设置、局限性、复用笔记
- 💬 论文问答：基于命中 chunk 生成带出处回答
- 🤖 聊天机器人：支持通用聊天，也支持基于选中文献进行 RAG 对话
- 🔐 本地优先：默认启用 heuristic 模式，也支持本地 `Ollama` 与外部 API 模型
- 📦 桌面打包：已支持生成 macOS `zip` 与 `dmg`

## 🏗️ 产品形态

Briefly AI 是本地软件。

- 文献文件解析优先在本机完成
- 索引与检索默认保存在本地
- 模型调用支持三种模式：
  - `heuristic`：零配置，本地启发式摘要和问答
  - `ollama`：连接本地模型进行摘要、RAG 与聊天
  - `api`：导入兼容 OpenAI 协议的 API，自主选择模型

## 🔌 API 导入能力

新增了类似 CC Switch 的“模型接入中心”，支持把外部 API 作为模型源导入到 Briefly AI。

目前支持导入兼容 OpenAI API 的服务，用户可以粘贴以下任意一种配置格式：

1. JSON 配置
2. 带查询参数的链接 / deep link 风格配置
3. `KEY=VALUE` 的环境变量风格配置

典型字段包括：

- `name`：提供商名称
- `baseUrl`：接口地址，例如 `https://api.openai.com/v1`
- `apiKey`：访问密钥
- `defaultModel`：默认模型

导入成功后，应用会自动拉取模型列表，用户可以在界面里选择当前激活的 Provider，并刷新模型清单。

## 🧭 模型路由

你可以把不同能力分别路由到不同模型：

- ✍️ 摘要模型：用于 TL;DR、方法总结、创新点等阅读辅助
- 📚 RAG 模型：用于论文问答和基于命中段落的回答生成
- 💬 聊天模型：用于通用聊天机器人或带文献上下文的对话

这让用户可以按成本、速度、上下文长度和能力强弱自由组合模型，而不是被单一模型绑定。

## 🧪 RAG 与聊天机器人

Briefly AI 当前支持：

- 将论文按章节切分为 chunks
- 建立本地轻量索引并执行混合检索
- 在问答结果中返回引用出处
- 使用选中文献作为上下文进行聊天
- 在聊天界面切换是否启用 RAG

适合的使用场景包括：

- 快速判断论文是否值得精读
- 回答“某篇论文的方法细节在哪里”
- 对同一主题文献做概念澄清和复述
- 把长论文阅读变成可追问、可回看的对话过程

- `dist/Briefly AI-0.1.0-arm64-mac.zip`

## 🏠 本地模型模式

默认模式不依赖外部模型。

如果想启用更强的本地问答、摘要与聊天能力，可先启动 Ollama，然后在应用左侧配置：

- `Ollama Base URL`：默认 `http://127.0.0.1:11434`
- `Model`：例如 `qwen3:4b`

## 🔒 数据与隐私

- 应用运行时数据默认存放在 Electron 的 `userData` 目录下
- PDF 解析、索引与基础检索可以完全本地完成
- 如果选择 API 模式，只有模型调用请求会发送到对应服务
